資料結構 LIFO O(1) 操作

1. Stack 如何運作

Stack 是後進先出（LIFO, Last In First Out）的線性結構： 新增與移除都只在同一端（top，頂端）進行。 想像一疊盤子——最後放上去的，會最先被拿走。

push（壓入）
從頂端放入
pop（彈出）
從頂端取出

互動試試看

top → （空）

觀察：

  • push 永遠加在最上層（top）。
  • pop 永遠移除最上層；空堆疊時為 underflow。
  • peek 只看頂端，不移除。
  • 所有操作皆為 O(1)。

2. 核心操作

操作說明時間複雜度
push(x)將 x 壓入頂端O(1)
pop()移除並回傳頂端元素O(1)
peek() / top()回傳頂端元素但不移除O(1)
isEmpty()是否為空O(1)
size()元素數量O(1)

3. Stack 的運用

函式呼叫堆疊

程式執行時的 call stack 保存返回位址與區域變數；遞迴即靠它運作。

復原 / 重做

編輯器的 Undo/Redo 以兩個 stack 分別記錄操作歷史。

瀏覽器上一頁

瀏覽歷史用 stack 實作「上一頁 / 下一頁」。

括號配對

遇左括號 push、遇右括號 pop 比對，檢查 ()[]{} 是否平衡。

運算式求值

後序式 (postfix) 求值、中序轉後序，皆以 stack 為核心。

DFS / 回溯

深度優先搜尋、迷宮回溯用 stack 記錄路徑（或以遞迴隱含使用）。

進位轉換

十進位轉二/十六進位時，餘數 push 後再 pop 即得正確順序。

語法剖析

編譯器以 stack 進行運算子優先序與語法分析。

4. 運算式表示法：Infix / Postfix / Prefix

同一個運算式可用三種方式書寫，差別在運算子相對於運算元的位置：

表示法運算子位置範例（A + B）需要括號/優先序？
Infix 中序在兩運算元中間A + B需要（人類習慣）
Postfix 後序（逆波蘭 RPN）在運算元之後A B +不需要
Prefix 前序（波蘭式）在運算元之前+ A B不需要

較複雜的例子

InfixPostfixPrefix
A + B * CA B C * ++ A * B C
(A + B) * CA B + C ** + A B C
A + B * C - D / EA B C * + D E / -- + A * B C / D E

為什麼要 Postfix/Prefix？ 因為它們不需要括號與運算子優先序規則，電腦只要一個 stack 就能直接、明確地求值， 非常適合機器處理。中序則符合人類閱讀習慣。
